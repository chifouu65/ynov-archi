# LIVRABLE SÉANCE 8 : Design Patterns Structurels

**Document :** Facade & Adapter
**Date :** Séance 8
**Statut :** Réalisé

## 1. Objectif : Simplifier l'accès et l'intégration

L'objectif de cette séance est double :
1.  Simplifier l'utilisation du système complexe via une **Facade**.
2.  Permettre l'intégration de composants ou systèmes externes via un **Adapter**.

## 2. Implémentation du Pattern Facade

Nous avons créé la classe `SystemFacade` (`com.jad.SystemFacade`) pour masquer la complexité des interactions entre `Factory`, `FooBuilder` et les autres entités.

### 2.1. Rôle
La Facade offre un point d'entrée unique et simplifié pour les clients qui n'ont pas besoin de connaître les détails de l'instanciation ou de la configuration fine.

### 2.2. Exemple d'utilisation
```java
// Au lieu de manipuler Factory et Builder séparément :
Foo system = SystemFacade.getInstance().createComplexSystem(bar, qux, bazList);
```

## 3. Implémentation du Pattern Adapter

Nous avons mis en place un **Adapter** pour permettre à notre système (`Foo`) d'être piloté comme un système externe standardisé (`ExternalSystem`).

### 3.1. Composants
*   **Target (`ExternalSystem`)** : L'interface attendue par le client externe.
*   **Adaptee (`Foo`)** : Notre classe métier existante.
*   **Adapter (`ExternalSystemAdapter`)** : La classe qui fait le pont. Elle traduit l'appel `performAction()` de l'interface externe en un appel métier interne (`foo.addGrault()`).

### 3.2. Bénéfice
Cela permet d'intégrer notre application dans un écosystème plus large sans modifier le code existant de `Foo`.

## 4. Livrables

*   ✅ **Accès unifié** : `src/main/java/com/jad/SystemFacade.java`
*   ✅ **Intégration externe** : `src/main/java/com/jad/ExternalSystemAdapter.java`
